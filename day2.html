Q1 What do you put in the Rows zone of a pivot table?
The field you want to group by — each unique value in that column becomes a row in your pivot table. For example, if you put county in Rows, you get one row per county.
Q2 What does COUNT do in the Values zone — what is it actually counting?
COUNT counts the number of rows in the dataset that belong to each group. It doesn't add up a number — it tallies how many records match. So if you group by county and COUNT, you get the number of crashes per county.
Q3 What's the difference between a filter and a row grouping?
A row grouping shows all categories as separate rows — you can compare them. A filter limits which records are included before grouping — it narrows the data so you're only looking at a subset. Filters apply first, then grouping happens on what's left.
Today's two new skills:
Filters — limit your data to a specific subset before aggregating.
Cross-tabs (Rows + Columns) — group by two categorical variables at once to see patterns across both.

Lesson 1 — Pivot filters

What is a pivot filter?

A filter narrows the data before the grouping happens. Think of it like a WHERE clause or an if-statement that sits in front of your loop. Only matching rows get grouped and counted.

Without a filter: Group all 500 crashes by county → shows totals for every county.
With a filter (Severity = Fatal): First keep only fatal crashes, then group by county → shows which county has the most fatal crashes specifically.

The result is completely different — and answers a much more specific question.

How to add a filter in Google Sheets
  1. Open your pivot table editor (click anywhere in the pivot table).
  2. In the editor panel on the right, find the Filters section and click Add.
  3. Choose the field you want to filter on (e.g. crash_severity).
  4. Click the filter dropdown — uncheck Select all, then check only the value(s) you want (e.g. Fatal).
  5. Your pivot table will instantly update to show only those rows.
Common mistake: Adding the filter field to Rows instead of Filters. If you put crash_severity in Rows, you see all severities as separate rows — you're not filtering, you're grouping. Filters go in the Filters zone.

Lesson 2 — Cross-tabs (Rows + Columns)

What is a cross-tab?

A cross-tab puts two categorical variables in the same pivot table — one as Rows and one as Columns. Each cell shows the aggregated value for that combination.

Example: Rows = county, Columns = crash_severity.
Each cell answers: "How many crashes in this county had this severity?"

You can scan across a row to see how severity breaks down for one county, or scan down a column to compare counties for one severity.
When to use it: Cross-tabs work best when you're comparing how two categories relate — not just "which is biggest" but "does the pattern look the same across groups?"
How to add a Columns field in Google Sheets
  1. In the pivot table editor, you already have a Rows field set.
  2. In the Columns section, click Add.
  3. Choose a second categorical field (e.g. crash_severity).
  4. Your table will expand to show each severity as a separate column header.
  5. Keep Values set to COUNT — you'll see a count for each row+column combination.
Warning: If your Columns field has many unique values (like city — 30+ cities), the table becomes unreadably wide. Stick to fields with 5–8 unique values for Columns.

My Finding — data storytelling

Explore the dataset on your own. Find one thing that surprises you. Then write it up as a clear, specific finding a non-technical reader could understand.

What makes a good finding?
  • Specific — names a number, a place, or a category. Not "there were lots of crashes."
  • Surprising or non-obvious — something you wouldn't have guessed without the data.
  • Supported by a pivot table — you can show the table that proves it.
  • In plain language — written for a local news reader, not a data scientist.

Example finding (model)

"Motorcycles made up only 4% of all crashes in Wayne County, but accounted for 21% of fatal crashes — more than five times their expected share. This suggests that motorcycle crashes are disproportionately deadly even when they happen less often."
