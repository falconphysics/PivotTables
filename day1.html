The data problem

Look at those 500 rows. If your teacher asked "which county has the most crashes?" — how would you find out from raw data alone?

This is what pivot tables solve. They automate the groupBy + aggregate operation — the same logic you'd write as a loop in Java. Today you'll learn to do it in seconds using Google Sheets.

Step 1 — Download the dataset

Download the CSV file below. A CSV (Comma-Separated Values) file is just a plain text file where each row is a crash record and each column is separated by a comma. Google Sheets can open it directly.

Step 2 — Open in Google Sheets
  1. Go to sheets.google.com and click + Blank to create a new spreadsheet.
  2. Click File → Import in the top menu.
  3. In the import dialog, click the Upload tab, then drag your CSV file into the window (or click to browse).

Check your import: Row 1 should be your column headers (crash_id, date, year, …). If the header row is missing or the data looks jumbled, try importing again and verify "Comma" is the separator type.

Build a pivot table — anatomy and steps

What a pivot table actually does

A pivot table takes raw tabular data and groups rows by a category, then summarizes a value for each group. In Java terms:

// What a pivot table does, translated to pseudocode:
HashMap<String, Integer> counts = new HashMap<>();
for (CrashRecord r : data) {
  String key = r.getCounty(); // ← the Rows field
  counts.put(key, counts.getOrDefault(key, 0) + 1); // ← COUNT
}
// Then sort counts by value descending

Pivot tables do this automatically — no code required. But now you know exactly what's happening under the hood.

How to insert a pivot table in Google Sheets
  1. Click any cell inside your data (e.g. cell A1).
  2. In the top menu, go to Insert → Pivot table.
  3. A dialog will ask "Insert to" — choose New sheet. Click Create.
  4. A new tab opens (called "Pivot Table 1") with the pivot table editor panel on the right side.
  5. In the Rows section, click Add and choose county.
  6. In the Values section, click Add and choose crash_id. Change "Summarize by" to COUNTA.
  7. Your first pivot table appears — a count of crashes per county.
Sort it: In the Rows zone, click the dropdown arrow next to "county" → Sort by → COUNTA of crash_id → Descending. Now the most common county floats to the top.
The aggregation options explained

When you set a Values field, you choose how to summarize the numbers. The most common options:

COUNTA
Count of non-empty cells — how many crashes are in each group. Works on any column.
SUM
Add up a numeric column — use on injuries, fatalities, or vehicles_involved.
AVERAGE
Mean of a numeric column — e.g. average vehicles involved per group.
MAX / MIN
Highest or lowest value in the group. Useful for spotting outliers.
Common mistake: Using SUM on a text column (like county or vehicle_type). Sheets sums nothing and shows 0. Always use COUNTA to count rows, not SUM, when your Values field is categorical.

Practice — build these in Google Sheets

Build each of these pivot tables in your Google Sheets file. Write your answer in the box below each question. Each question tells you exactly which zones to use.

Q1 Which vehicle type was involved in the most crashes?
rowscount
▾

Setup: Rows = vehicle_type · Values = COUNTA of crash_id · Sort descending

Insert a new pivot table. In the editor, click Add next to Rows → choose vehicle_type. Click Add next to Values → choose crash_id → Summarize by COUNTA. Then in the Rows section, click the dropdown → Sort by → COUNTA of crash_id → Descending.
Q2 Which county had the highest total number of injuries?
rowssum
▾

Setup: Rows = county · Values = SUM of injuries · Sort descending

Create a new pivot table (Insert → Pivot table → New sheet). Rows = county. Values = injuries, Summarize by SUM. Sort by SUM of injuries descending. This is different from Q1 — you're adding up a number, not just counting rows.
Q3 What was the most common contributing factor across all crashes?
rowscount
▾

Setup: Rows = contributing_factor · Values = COUNTA of crash_id · Sort descending

Same process as Q1 — just change the Rows field to contributing_factor. After sorting, the factor at the top of the list is the most common. Note that "Unknown" may appear near the top — this is a data quality issue, not a real factor.
Q4 Which time of day had the most crashes? Does this surprise you?
rowscount
▾

Setup: Rows = time_of_day · Values = COUNTA of crash_id · Sort descending

Rows = time_of_day. Values = COUNTA crash_id. Sort descending. You'll see 5 rows — one for each time window. Think about why one time of day might dominate. Consider how many hours are in each window vs. how many cars are on the road.
Q5 Which crash severity category had the highest average number of vehicles involved?
rowsaverage
▾

Setup: Rows = crash_severity · Values = AVERAGE of vehicles_involved · Sort descending

Rows = crash_severity (4 values). Values = vehicles_involved, Summarize by AVERAGE. Sort descending. You're now asking a different question — not which severity is most common, but which severity tends to involve more vehicles per crash.
