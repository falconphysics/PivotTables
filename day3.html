Advanced pivot skills — going deeper

Day 1 covered grouping and counting. Day 2 added filters and cross-tabs. Today you'll add three new techniques — then turn your results into charts.

A Calculated fields — percentages and ratios

A calculated field lets you define a custom formula that runs on the aggregated values in your pivot table — not on the raw data. The most useful one: turning counts into percentages.

Example goal: What percentage of crashes in each county were fatal?

You can't just COUNT fatalities — you need fatalities ÷ total crashes per county.
In Google Sheets, add a helper column next to your pivot table:
= B2 / SUM($B$2:$B$11) — where B2 is the count for one row and $B$2:$B$11 is the full Values column.

There's a second approach: add two value fields to the same pivot table — one COUNTA for total crashes, one SUM for fatalities — then calculate the ratio in a helper column alongside.

Sheets tip: After building the pivot table, click an empty cell in the same row (column C or D). Type =B2/GETPIVOTDATA("COUNTA of crash_id", $A$1) — or simply reference the cells directly. Format the helper column as a percentage: Format → Number → Percent.

B Year-over-year comparison

A cross-tab with year in the Columns zone turns a single-variable pivot into a year-over-year comparison. You can immediately see whether a category is trending up or down.

Setup: Rows = contributing_factor · Columns = year · Values = COUNTA of crash_id

You'll get a table where each row is a factor and each column is a year (2022, 2023, 2024). Reading across a row tells you whether that factor increased or decreased over time.
Watch out for unequal time windows. If your dataset has more data from one year than others, raw counts aren't comparable. Always check whether the totals per year are similar before drawing trend conclusions.

C Nested row groups

You can add multiple fields to the Rows zone — the second field nests inside the first. This creates a hierarchy: group A contains group B.

Example: Rows = county, then road_type · Values = COUNTA

Result: each county expands into sub-rows showing how its crashes break down by road type. You can see that Wayne County's crashes concentrate on City Streets, while Livingston's are more spread across highways.
  1. Build a pivot table with Rows = county.
  2. In the Rows section, click Add again and choose road_type.
  3. Google Sheets automatically nests road_type inside county — each county row expands into road type sub-rows.
  4. Click the triangle arrows to collapse/expand individual counties.

Chart types — choosing the right one

Not every chart type works for every question. The wrong chart can hide the pattern or actively mislead. Here's how to choose.

The decision framework
BAR CHART
Comparing categories side by side. Best for ranked pivot results.
→ crashes per county
→ top contributing factors
STACKED BAR
Comparing totals AND composition at the same time. Best for cross-tab results.
→ severity by county
→ vehicle type by year
LINE CHART
Showing change over time. Best for year, month, or time-of-day data.
→ crashes per month
→ year-over-year trends
PIE / DONUT
Showing part-of-whole. Only works with 5 or fewer categories.
→ crash severity share
→ road type breakdown
Pie chart trap: Never use a pie chart with more than 5–6 slices. With 11 vehicle types, a pie chart is unreadable. Use a bar chart instead.
Interactive comparison — same data, four chart types

Use the buttons to switch between chart types on the same pivot result (crashes by crash severity). Notice how each one emphasizes different things.

When to use stacked vs. grouped bars

Both use bars with multiple colors. The difference is what question they answer.

Stacked = "What is the total, and how is it composed?" — good for seeing that Wayne has far more crashes overall, and that most are property damage.

Grouped = "How does each severity compare across counties?" — good for seeing that Wayne has more of every severity, not just one.

Charts in Google Sheets — step by step

How to create a chart from a pivot table
  1. Go to your pivot table tab. Click any cell inside the pivot table results (not the editor panel — click the actual data).
  2. Select the data you want to chart. For a simple bar chart of county counts: click the county column header, then hold Ctrl (or Cmd on Mac) and click the Values column header. Both columns should be highlighted.
  3. Go to Insert → Chart. Google Sheets will guess a chart type — it's often wrong. You'll fix this in the next step.
  4. In the Chart editor that opens on the right, change Chart type to the one you want (Column chart for vertical bars, Bar chart for horizontal).
  5. Under Customize tab: add a chart title, axis labels, and adjust colors. A chart without a title is unusable.
  6. Click outside the chart editor to close it. Your chart is now embedded in the sheet.
Pro move: Right-click the chart → Move to own sheet. This gives your chart its own full-size tab, which is much easier to read and present.
Chart titles and axis labels — required, not optional

A chart without context is meaningless. Every chart you submit must have:

  • A title that states what the chart shows (not just "Chart 1"). Example: Michigan Traffic Crashes by County, 2022–2024
  • An X-axis label describing the categories
  • A Y-axis label describing the values and units (e.g. "Number of crashes")
  • A legend if there are multiple data series (e.g. one bar per year)
Watch the scale. Google Sheets sometimes starts the Y-axis at a non-zero value, which can make small differences look enormous. Check: Chart editor → Customize → Vertical axis → Min value → set to 0.
Making a stacked bar from a cross-tab
  1. Build a cross-tab: Rows = county · Columns = crash_severity · Values = COUNTA.
  2. Select all the data in the pivot result — county names, severity column headers, and all the counts.
  3. Insert → Chart. In Chart type, choose Stacked column chart.
  4. Sheets should automatically use county as the X-axis and each severity as a separate color series. If it doesn't, go to Setup → X-axis and manually set it.
  5. Filter to just the top 5–6 counties first to keep the chart readable — too many bars is hard to interpret.
Color tip: In the Customize tab, assign meaningful colors: Fatal = dark red, Serious Injury = orange, Minor Injury = yellow, Property Damage = green. Colors that encode severity create immediate visual hierarchy.
Making a line chart for time trends
  1. Build a pivot table: Rows = month · Values = COUNTA of crash_id. This gives crashes per month (all years combined).
  2. For year-over-year: Rows = month · Columns = year · Values = COUNTA. Now you have 3 lines — one per year.
  3. Select all the data. Insert → Chart → Line chart.
  4. The X-axis should be month. If Sheets puts year on the X-axis instead, go to Setup and swap the rows/columns toggle.
Month ordering problem: Sheets sorts months alphabetically (April, August, December…) instead of chronologically. Fix this: in your raw pivot table, add a helper column with month numbers (January=1, February=2, …) and sort by that before charting.

Q1 What percentage of each county's crashes were fatal? Create a bar chart showing fatality rate per county.
calculated %bar chart
▾

Pivot setup: Rows = county · Values = COUNTA crash_id AND SUM fatalities (add both). Then in a helper column: = fatalities / total. Chart: bar, sorted descending by fatality rate. Title: "Fatal Crash Rate by County, 2022–2024".

Add two value fields to the same pivot: first COUNTA of crash_id (total crashes), then SUM of fatalities. Your pivot now has two value columns. In column D, type =C2/B2 (fatalities ÷ total) and drag it down. Format as percentage. Build your bar chart from columns A and D only.
Q2 How did the top 5 contributing factors change year over year? Build a grouped bar chart.
year-over-yeargrouped bar
▾

Pivot setup: Rows = contributing_factor · Columns = year · Values = COUNTA. Filter to top 5 factors. Chart: grouped column chart (each year = one color). Title: "Contributing Factors by Year, 2022–2024".

Build the cross-tab. Then manually select only the rows for your top 5 factors (hold Ctrl and click each row). Insert chart. In the chart editor Setup tab, check that "Use row 1 as headers" and "Use column A as labels" are both on. Switch to Grouped column chart.
Q3 What does crash severity look like by vehicle type? Build a stacked bar chart.
cross-tabstacked bar
▾

Pivot setup: Rows = vehicle_type · Columns = crash_severity · Values = COUNTA. Chart: stacked column chart. Assign severity colors manually (dark red = Fatal, etc.). Title: "Crash Severity by Vehicle Type".

The cross-tab gives you each severity as a separate column — exactly what you need for a stacked bar. Select the whole pivot result, insert chart, choose Stacked column chart. In Customize → Series, click each severity label and assign your chosen color.
Q4 How do crash counts change across the months of the year? Build a line chart.
time trendline chart
▾

Pivot setup: Rows = month · Columns = year · Values = COUNTA. Fix month ordering by adding a sort helper column. Chart: line chart (one line per year). Title: "Monthly Crash Counts by Year".

After building the pivot, the months will be in alphabetical order. Add a helper column in your pivot sheet with the month number (Jan=1, Feb=2, etc.) and sort ascending by that column. Then select month name + the three year columns → Insert chart → Line chart.
